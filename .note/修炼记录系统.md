# 修炼记录系统

## 架构
- **后端**: Node.js + Express + MySQL (mysql2/promise)
- **前端**: 原生 HTML/CSS/JS 单页应用
- **认证**: JWT (30天有效期)
- **数据库**: MySQL (远程 36.138.73.49:3377, 库名 xiulian)

## 关键文件
- `server.js` - Express 服务器，所有 API 路由（全部 async/await）
- `db.js` - MySQL 连接池，query()/execute() 辅助函数
- `public/index.html` - SPA 主页面
- `public/style.css` - 暗色玻璃态主题
- `public/app.js` - 前端逻辑

## 评分逻辑（重要）
- **打坐分(50分)**：≥30分钟满分，不足按比例
- **起床分(25分)**：≤9:00满分，每超20分钟扣剩余30%（含不满20分钟的按比例）
- **睡觉分(25分)**：≤0:30满分，每超20分钟扣剩余30%
- **总分 = 打坐分 + 起床分 + 睡觉分**

## 注意事项
- 使用 `mysql2/promise` 连接池，所有数据库操作异步
- `query(sql, params)` 返回行数组，`execute(sql, params)` 返回 result 对象（含 insertId）
- 日期使用 Asia/Shanghai 时区（前端 `new Date()` 取本地时间，MySQL 连接池配置 timezone: '+08:00'）
- 睡觉时间判定：18:00~0:30 为正常，0:30~6:00 为超时

## 排行榜用户详情
- 点击排行榜中任意用户条目 → 弹出弹窗展示该用户的每日详细记录
- API：`GET /api/user-detail?username=xxx&range=day|week|month|halfyear&date=yyyy-mm-dd`
- 弹窗包含：顶部汇总统计 + 每日卡片（打坐记录/作息详情/凡心记录完整内容）
- 日榜始终显示所有天，其他时段过滤无活动天

## 感悟广场
- 所有用户可发布修炼感悟，互相浏览
- 表：`insights`（id, user_id, content, created_at）
- API：
  - `GET /api/insights?page=1` — 分页查询，每页20条，按时间倒序，JOIN users 获取用户名
  - `POST /api/insights` — 发布感悟，body: `{ content }`
  - `DELETE /api/insights/:id` — 删除自己的感悟
- 前端：底部导航第6个tab（💡 感悟），卡片展示用户头像首字母+用户名+相对时间+内容
- 只能删除自己的感悟，"加载更多"分页
